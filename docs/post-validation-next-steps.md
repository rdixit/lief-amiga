**Source:** Combined Validation D/F Internal Expert Validation Report  
**Date:** 2026-05-18  
**Status:** Draft for review

## 1. Vocabulary Gaps to Address

The validation report recommends that several repair/recovery phrases remain fast and accessible. An audit of `vocabulary.json` reveals three gaps:

| Phrase | Status | Action Needed |
|--------|--------|---------------|
| food | Tier 1, present | None |
| water | Tier 1, present | None |
| bathroom | Tier 1, present | None |
| help me | Tier 1, quick phrase | None |
| I'm frustrated | Tier 2, quick phrase | None |
| I need a break | Tier 1, quick phrase | None |
| I don't want that | Tier 2, quick phrase | None |
| no | Tier 1, Negation/Repair | None |
| stop | Tier 1, Actions | None |
| **different** | **Tier 3, Descriptors** | **Promote to Tier 1 or 2** — called out as a core repair word |
| **not that** | **Missing** | **Add as quick phrase or Tier 1 phrase** — key repair/refusal expression |
| **try again** | **Partial (Tier 3 words only)** | **Add as combined phrase, Tier 1 or 2** — important repair pathway |
| **I meant something else** | **Missing** | **Add as phrase, Tier 2** — supports communication repair |

### Priority actions:
- Add "not that" as a quick phrase and/or Tier 1 symbol with repair intent tags
- Promote "different" from Tier 3 to Tier 1 (Negation/Repair category)
- Add "try again" as a combined phrase (Tier 1 or 2)
- Add "I meant something else" as a Tier 2 phrase with repair intent tags

## 2. Validation E — Vocabulary Expansion and Category Design

The report identifies Validation E as the next iterative step. This is a lightweight SLP vocabulary and category review, not a full ontology task.

### Recommended review format:

| Column | Purpose |
|--------|---------|
| Word / Phrase | What should be added |
| Suggested Category / Placement | Where users would expect to find it |
| Tier 1–3 | How urgently it should be accessible |
| Notes / Typical Use | Context, alternate placement, or concerns |

### Key design questions for Validation E:
- Which words belong in Tier 1 (immediate access)?
- Which words should appear in multiple places (cross-listed)?
- Which phrases must be one-hit or top-bar accessible?
- Which categories are intuitive to both SLPs and likely users?
- Does the layout remain usable as vocabulary grows?

## 3. Preserve and Protect Repair Pathways

Repair/refusal should remain a standing validation scenario in every future validation phase. It is the clearest test of whether AMIGA can recover from prediction errors or missing vocabulary.

### Recommended standing repair test prompts:
- "Not that"
- "I don't want that"
- "Different"
- "Stop"
- "Try again"
- "I meant something else"

### Design principle:
If repair phrases are not immediately accessible, prediction errors become communication dead ends. Repair words must remain highly visible and rapidly accessible as vocabulary scales.

## 4. Grid View and Meaning Room — Complementary Modes

The validation found that grid and room views serve different interaction needs:

| View | Role |
|------|------|
| Grid view | Stable access, category learning, repeatable navigation |
| Meaning Room | Meaning-based discovery, visual scaffolding, engagement-oriented access |

### Recommendation:
- Continue supporting both access modes
- Do not force a premature choice between them
- One evaluator preferred grid; both saw value in room view for some users
- Room view should be treated as a promising complement, not the sole primary pathway

## 5. Reduce Literacy Dependence

Evaluator feedback noted that ease of use may depend on the evaluator's literacy and familiarity, and that many target users (students) may not experience the same ease.

### Areas to strengthen:
- Icon clarity and consistency
- Symbol-to-meaning correspondence
- Visual affordances for non-readers
- Non-text access pathways
- Speech output reliability

## 6. Usability Metrics — Continue with Caution

Continue using SUS and NASA-TLX for comparing prototype iterations, but interpret them alongside:
- Scenario success and functional support ratings
- Cognitive load measures
- SLP qualitative notes
- In-app interaction metrics (when available)
- Future participant data (post-IRB approval)
